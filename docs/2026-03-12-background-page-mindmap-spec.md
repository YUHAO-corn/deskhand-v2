# Background 页改版开发需求

## 背景

Background 页目前是纯文字（5 段话，居中单栏）。这次改版不是简单地把页面拆成“左文右图”，而是要把这页做成一个**统一画面**。同时有几个前提不能动：

- 现有 5 段文案内容不改
- 文案仍然要保留“完整自述”的阅读感
- 右侧思维导图结构仍按本文档定义的完整树，不做 3-4 个主簇的收敛版
- 思维导图只能辅助这段自述成立，不能把它打碎成说明文 + 插图

因此，这页的目标不是“介绍功能 + 配一张图”，也不是桌面端做成“上文下图”，而是：

- 文案负责提出问题和建立叙事入口
- 思维导图负责展示 Deskhand 作为完整产品的功能全貌
- 两者在同一张画布里共同构图，但文案的连续阅读优先级高于导图

这一页最终要让读者同时感受到两件事：

1. Deskhand 不是单点功能，而是一个完整的桌面智能体产品
2. 后面要展开讲的三个能力，是完整产品中的三个亮点，而不是三个孤立 feature

---

## 这份 spec 解决的两个问题

### 1. 页面必须有整体构图，不能是“文字 + 插图”的拼装

原方案的问题是把 prose 和 mind map 当成两个并列模块处理，容易做成“左边已有一段文案，右边再塞一张图”的效果。这样会破坏这一页应该有的整体性。

这次的设计原则改为：

- 页面是一个统一的 scene，不是左右两个独立容器
- 文案块是画面的叙事中心，不是单独的一栏
- 思维导图的主干线要围绕文案块组织，但不能破坏文案的完整朗读感
- 根节点、文案块、分支群三者共同决定页面重心，其中优先保护文案块

### 2. 思维导图不能用框选式节点，而要用连续曲线型样式

不采用“节点外框 + 分叉短线 + 外置徽标”的常规脑图样式。改用用户指定的方向：

- 所有节点默认都是纯文字
- 主干线和分支线都是连续、自然、顺滑的曲线
- 不为一级节点加矩形边框
- 高亮分支的标记和标题是一体的，不做外置 badge
- 整体更像“从一个根节点自然生长出来的结构图”，而不是流程图或知识树

## 整体构图

### 构图原则：单一画布，且叙事优先

桌面端不要先做一个 38 / 62 的两栏 grid，再把内容分别放进去。正确做法是：

- 最外层仍使用一个 `SectionCanvas`
- 内部建立一个 `position: relative` 的 unified stage
- prose block 和 mind map 都放在这个 stage 内，通过绝对定位和层级关系共同构图
- prose block 必须保持为一个完整、连续、可一口气读完的文字块
- 思维导图 SVG 可以覆盖整个 stage，但必须为 prose block 留出完整安全区，避免线条和节点破坏自述感
- “叙事优先”指的是阅读优先级，不是桌面端的上下排布顺序

### 画面结构

```text
┌────────────────────────────────────────────────────────────────────┐
│   [ prose / narrative block ]  │  [ Deskhand mind map ]          │
│   完整自述，不被图形切碎或打断   │  连续曲线、轻量、安静地存在       │
│                                                                    │
│  [ root label ] ──╮        curve around prose                      │
│                   ├─────────────── [ topics ]                      │
│                   ╰─────────────── [ topics ]                      │
│                                                                    │
│                         [ 3 个高亮分支穿插其间 ]                    │
└────────────────────────────────────────────────────────────────────┘
```

### 桌面端布局建议

桌面端不是上下布局，而是**左文右图的一体化布局**。这里的“一体化”不是说两边互相穿插，而是说它们在同一张画布里通过比例、留白和视觉节奏形成统一关系。

- prose block 位于左侧，宽度控制在整页 `28% - 34%`
- prose block 内部不插入任何图形元素、标题标签或高亮标注，保持它像一段完整口述
- prose block 与 mind map 之间保留一条很轻的纵向分隔线，帮助稳定左右关系
- 根节点位于右侧画布偏左的位置，作为整张导图的起笔点
- 思维导图主要在右半区展开，占据中部到右侧的大部分空间
- 上下分支拉开，给导图留足呼吸感，避免右侧太拥挤
- 左右两块内容通过统一基线、统一留白和统一色调形成一个完整画面，而不是通过元素交叠来制造“一体”

可以理解为：

- 左侧是一段完整自述
- 右侧是一张收束得当的产品结构图
- 中间用轻分隔和大留白建立秩序
- 两者风格一致，所以是一体的

而不是：

- 上面一整块长文
- 下面再摆一整张图

### 建议的空间分布

以 16:9 桌面视口为目标，优先按相对比例控制：

- prose block：`x 4% - 8% / y 18% - 26% / width 24% - 30%`
- divider：位于整页 `x 30% - 33%`，高度不必顶满全屏，透明度较低
- root label 锚点：`x 34% - 40% / y 48% - 58%`
- branches 活动区：`x 38% - 95% / y 14% - 88%`
- prose 安全区：导图路径和节点不得穿入 prose block 外扩 `32px` 的区域

这几个区块的目的不是死板套坐标，而是确保开发时始终把这页当成一个整体画面来摆。

### 文案区处理

左侧文案的内容本身不改，但视觉和排版需要为整幅画面服务：

- `kicker`、段间距、字重、行高都可以调整
- 可以通过更稳的段落节奏和更明确的最后一句收束，让文案与右侧导图形成呼应
- 最后一段允许做轻微文案调整，以便自然过渡到右侧导图，例如先点出“整体如右”，再接“接下来讲其中三个亮点”

重要限制：

- 前四段内容不改写
- 不把正文拆成小标题 + bullet
- 不把正文改成说明式导读

### 移动端退化

移动端（`< 920px`）再退化成单栏：

- prose 在上
- mind map 在下
- 根节点缩小
- prose 仍作为一个完整文字块先被读到
- 一级分支数量不减少，但子节点可以适度压缩间距

### 文案与图的关系

这一页的阅读顺序必须是：

1. 先读完整段自述
2. 再把视线移到导图，理解“原来这是一个完整产品”
3. 最后自然接受“后面会展开讲三个亮点”

也就是说，导图的职责不是解释每一句文案，而是在读完自述之后，补上一层产品全貌认知。

这里再强调一次：

- “先读文案，再理解导图”说的是视线和认知顺序
- 不代表视觉上必须把导图放在文案下面
- 桌面端必须保持横向整合的单画布关系

## 思维导图内容

### 根节点

```text
Deskhand
桌面智能体客户端
```

说明：

- 根节点按品牌优先处理，主标题使用 `Deskhand`
- 在其下方补一行较小的功能说明，如“桌面智能体客户端”
- 根节点需要像草图那样成为右侧导图的中心锚点，而不是纯概念词标签

### 一级分支结构

普通分支（8 个）：

```text
对话与会话
  ├── 多会话管理
  ├── 会话持久化
  └── 流式输出

Agent 引擎
  ├── Claude Agent SDK
  ├── 工具调用
  └── 事件流

工具过程可视化
  ├── 活动树
  ├── 状态追踪
  └── 子代理嵌套

Artifact 面板
  ├── 代码预览
  ├── HTML 渲染
  └── 文件管理

文件与项目
  ├── 工作目录
  ├── 文件附件
  └── 本地存储

权限控制
  ├── Ask / Auto 模式
  └── 权限弹窗

MCP 集成
  ├── 连接管理
  └── 工具路由

设置与配置
  ├── 模型选择
  ├── API 配置
  └── 语言
```

亮点分支（3 个）：

```text
Generative UI
  ├── Playground
  ├── This or That
  └── Guided Form

Skill 系统
  ├── 内置 Skills
  ├── Skill Insight
  └── 一键安装 / 自定义创建

剪贴板感知
  ├── 实时监控剪贴板
  ├── 历史记录管理
  └── AI 主动读取调用
```

共 11 个一级分支，其中 3 个为高亮分支。

### 分支分布建议

不要把 3 个高亮分支堆在一起。建议穿插在普通分支之间，形成自然节奏：

- 上半区放 1 个高亮分支
- 中部放 1 个高亮分支
- 下半区放 1 个高亮分支

这样读者会感受到“完整产品图谱里自然浮现出的三个亮点”，而不是一个单独的 highlights 区。

## 视觉规格

### 视觉风格：连续曲线型脑图

参考用户给出的样式方向，最终效果应满足以下原则：

- 根节点到一级分支使用连续的主干曲线
- 一级分支到子节点继续自然延伸，不要断成短横线 + 文本标签
- 所有节点默认无边框、无底色、无卡片
- 线条本身就是版式骨架，承担组织画面的职责
- 导图整体视觉强度低于左侧文案，不要做成主视觉海报

### 节点样式

| 元素 | 规格 |
|------|------|
| 根节点 | 大字号、粗体、强调色，含一行较小副标题，作为右侧导图锚点 |
| 普通一级分支 | 纯文字，无边框，`font-weight: 600` |
| 高亮一级分支 | 与普通分支同结构，但通过颜色和字重略微强调 |
| 子节点 | 小一号文字，弱化处理，只保留必要信息密度 |

### 连线样式

| 元素 | 规格 |
|------|------|
| 根节点主干线 | 一笔连续拉出，线宽略粗于其他线 |
| 一级分支线 | 平滑 cubic bezier 曲线，避免锐角折线 |
| 子节点延展线 | 从一级分支末端顺势延伸，长度短、线更细 |
| 线条关系 | 同一分支内的主线和子线要有连续感，不能像两套系统 |

### 高亮分支规则

高亮不使用星标，直接靠分支本身完成：

- 通过分支颜色、线宽、字重和位置体现重点
- 高亮分支和普通分支仍属于同一视觉系统，不能额外贴 icon 或 badge
- 禁止使用外置星标、圆形徽章或悬浮装饰

### 色彩策略

- 不引入新的硬编码色值
- 使用现有 design tokens：`--ink`、`--ink-soft`、`--ink-muted`、`--sage`、`--line`
- 普通分支以 `--ink` / `--ink-muted` 为主
- 高亮分支和根节点优先使用 `--sage`
- 通过字重、透明度、线宽制造层次，不靠多彩配色
- 保留一条淡纵向分隔线，颜色同样来自现有 token，透明度要轻

## 实现方式

### 推荐实现：HTML prose + 分栏式 stage + SVG 导图

不要做桌面端上下堆叠。推荐结构：

1. `BackgroundSection` 内创建一个 `stage`
2. 左侧 prose block 作为 HTML 元素放入 stage，完整保留现有段落结构
3. 中间增加一条淡纵向分隔线
4. 右侧 `<MindMap />` 作为 inline SVG 放入导图区
5. SVG 中硬编码根节点、分支节点和路径坐标

这样做的好处：

- prose 依然用正常 HTML 渲染，能完整保留原文案和阅读节奏
- 导图能精确控制曲线和层次
- 视觉上是统一画布中的左右关系，不是“补一张插图”

### SVG 实现约束

- 使用单一 `viewBox` 管理整体布局
- 根节点、一级分支、子节点都放在同一坐标系统里
- 主路径优先使用 `<path d="M ... C ..."/>`
- 子节点连接线也用短曲线，不用生硬直线
- 文本和线条要成组组织，便于后续微调

## 验收标准

- [ ] 页面一眼看上去是一个整体画面，不是左右拼版
- [ ] 现有 5 段文案内容不改，且仍保持完整自述感
- [ ] 左侧文案允许优化视觉排版，最后一句允许轻微调整以衔接右侧导图
- [ ] prose 和导图之间存在明确的构图关系，而不是各占一半空间
- [ ] 根节点、prose block、分支群有清晰主次和视觉重心，且 prose 优先级最高
- [ ] 11 个一级分支全部展示，3 个亮点分支自然穿插其中
- [ ] 思维导图使用连续曲线，无节点边框
- [ ] 高亮不使用外置星标或 badge，只靠分支自身体现
- [ ] 颜色全部来自现有 CSS 变量，不硬编码新色
- [ ] 左右之间存在一条淡分隔线，帮助画面稳定
- [ ] 桌面端保持一屏内完成阅读
- [ ] 移动端退化后仍可读，不出现严重遮挡
