## 1. 这轮 review 的结论

这轮不是再补零碎样式，而是要把 V2 拉回“作品展示站”的表达边界。

核心问题集中在四类：

1. 顶部 CTA 不完整，而且命名没有告诉用户点击后会发生什么。
2. 第 5、7、8、10 页已经出现模板和内容不匹配的问题，继续硬塞会越来越乱。
3. 个别文案有明显 AI 痕迹，口吻不够自然。
4. 几张高比例图片没有按真实尺寸反推布局，导致裁切、压缩、截断同时出现。

一句话判断：

**下一步不该继续“哪里溢出修哪里”，而是补齐 CTA、修正文案口吻，并为高比例图片增加专门版式。**

## 3. 全局问题：Header CTA

对应原始反馈：

- 1. 右上角少了下载简历按钮
- 2. `Decision` 打开的不是预设页面，命名也不合适
- 3. CTA 文案应明确点击结果
- 14. 先放一个可替换的简历占位文件
- 15. GitHub 应直达项目仓库，而不是个人主页

### 当前判断

当前 header 的两个问题是同时存在的：

1. 信息不全：少了简历下载入口
2. 命名不清：`Decision` 是抽象名词，不是 CTA

对展示站来说，右上角按钮最好统一成“动作 + 目标”的结构，而不是一部分是品牌名、一部分是抽象概念。

### 推荐 CTA 方案

推荐直接改成下面三枚：

- `View Repo`
- `View Design Notes`
- `Download Resume`

对应跳转：

- `View Repo` -> `https://github.com/YUHAO-corn/Deskhand`
- `View Design Notes` -> `https://yuhao-corn.github.io/column/`
- `Download Resume` -> 临时先接 `/resume.txt`，后续正式替换为同位置的简历文件

### 为什么不继续用 `Decision`

`Decision` 的问题不在于英文，而在于它没有告诉用户：

- 这是外链还是站内跳转
- 点进去会看到文章、文档还是某一页 PPT

`column` 这个站本质上更像一组设计/架构记录，所以 `View Design Notes` 比 `Decision` 更准确，也更像一个真实 CTA。

如果想更短一档，备选可以是：

- `Read Notes`
- `Design Notes`

但第一推荐仍然是：

**`View Design Notes`**

### 修复动作

1. 将 GitHub 链接从个人主页改为项目仓库
2. 新增 `Download Resume` 按钮，并保留强按钮样式给它
3. 将 `Decision` 改为 `View Design Notes`
4. 简历文件短期先沿用现有 `public/resume.txt` 作为占位
5. 等正式简历准备好后，用同一路径或同一按钮配置替换

### 涉及文件

- `src/components/chrome/SiteHeader.tsx`
- `src/components/chrome/SiteHeader.module.css`

## 5. 第 5 页：高度、文案、推理链版式

对应原始反馈：

- 5. 推理念组件高度太高，与上方内容重叠
- 6. “一扇没有把手的门”这句太 AI
- 7. 左边推理链和右边结论拉太开，视觉奇怪

### 当前判断

这页的问题不是单点 bug，而是 `chain` 模板本身没有为长标题、双观察卡、双链路同时出现做压缩设计。

最影响观感的是这两件事：

1. 下方链路卡太高
2. 旧/新链路把“结论 note”单独拉到最右边，信息被扯散

### 推荐文案替换

把这句：

`Skill 生态对她来说是一扇没有把手的门。`

改成：

**`Skill 生态就在那儿，但她既找不到入口，也不知道第一步该怎么开始。难道他要像应用商店一样去下载吗？`**

这个版本更自然，也更贴近中文语感。

备选：

`Skill 生态就在那儿，但她既找不到入口，也不知道第一步该怎么开始。`

### 推荐版式调整

不要继续保留现在这类“标签 | 主链路 | 右侧结论”三栏结构。

推荐改成每行两列：

- 左侧：`旧 / 新` 标签
- 右侧：主链路 + 次级说明上下堆叠

也就是把现在最右边的 `note` 收回到主链路下面，变成同一组信息。

这样能同时解决：

- 视觉拉扯过大
- 高度浪费
- 结论和链路脱节

### 修复动作

1. 替换“没有把手的门”这句
2. 压缩上半区文案高度，避免和下方链路板块抢空间
3. 重做 `chainRow` 的栅格结构，不再使用固定右侧结论列
4. 适度下调链路卡内边距和假设文案占位

### 涉及文件

- `src/content/skill-system.ts`
- `src/components/sections/ProblemSplitSection.module.css`

你提到的“占位图其实就是正式资源”，应按正式资源处理，不要继续让真实交付文件保留 `placeholder` 语义。

当前已确认的本地 PNG 尺寸：

- `skill-system-architecture-placeholder.png`
- `1580 x 1166`
- 约 `1.36:1`

### 推荐方案

第 7 页和第 10 页应统一成同一类模板：

**diagram-focus：左侧完整展示架构图，右侧用 narrative rail 承载标题、lead 和工程决策。**

1. 左侧图片列稍宽，适配 `1.36:1` 的横向图
2. 右侧保留 4 条决策，但改成更紧凑的编号列表，不再使用 4 张厚卡片

这样能保证：

- 图片继续完整展示

## 10. 建议的实现顺序

为了避免另一窗口一上来就陷进 CSS 细修，建议按这个顺序做：

1. 先修 header CTA
   - 一次性改完 repo / notes / resume 三个入口
2. 再修内容文案
   - 第 2 页标题
   - 第 5 页观察文案
   - 第 9 页右图 caption
3. 再做高比例图片专用模板
   - 第 7 页
   - 第 10 页
4. 最后修剩余溢出和节奏
   - 第 5 页链路高度
   - 第 8 页底部留白

这样做的原因是：

- 先改文案，才能知道真实内容高度
- 先确认图片版式，才能知道还剩多少空间给决策卡
- 如果顺序反过来，前面的 CSS 很容易白调
