# 过渡页共识（Hero → Gen UI 之间）

## 为什么需要这一页

当前 deck 的问题：Hero 讲完"我做了一个产品"，下一页直接跳进 Gen UI 的具体问题。读者缺少两个东西：

1. 不知道作者是谁、为什么做这个（没有建立信任）
2. 不知道接下来要听什么（没有叙事预期）

三个章节（Gen UI、Skill System、Clipboard）各自内部逻辑通顺，但章节之间的关系是隐性的。读者要到倒数第二页（My Take）才知道它们指向同一个方向。

## 叙事角度

从自己出发，不是专家做分享报告，而是一个 AI 产品经理 + 非技术用户的真实经历。

归纳法，不是演绎法：先有经历 → 再有发现 → 最后才有总结。不在开头亮框架或下结论。

## 信息点拆分

**信息点 1：我是谁**
我是 AI 产品经理，本身就在这个领域里，同时我自己也不写代码——我就是那个"非技术用户"。

**信息点 2：为什么做**
我自己用 Claude Code 的时候就感觉到很多事做不顺，身边不懂技术的朋友更是完全进不来。所以我决定自己做一个桌面版本。

**信息点 3：做的过程中发现了什么**
真正开始做之后，站在非技术用户的角度重新审视 AI 交互，发现问题不只是"没有 UI"这么简单。

**信息点 4：过渡到章节**
轻点一下：有几个具体的地方是可以做得更好的，接下来一个一个讲。

## 整体叙事线（更新后）

1. **Hero** — 这是 Deskhand，一个桌面 AI 应用
2. **过渡页（新增）** — 我是谁、为什么做、做的过程中发现了什么、接下来讲什么
3. **Gen UI ×3** — 第一个发现：用户说不清需求，需要自然语言之外的交互
4. **Skill System ×3** — 第二个发现：Skill 生态对非技术用户不友好，需要 Agent 代劳
5. **Clipboard ×3** — 第三个发现：用户的工作上下文在剪贴板里，AI 看不见
6. **My Take** — 回头看，这三件事指向同一个方向（这里才亮框架）
7. **Why Desktop** — 而这些只有在桌面端才做得到
