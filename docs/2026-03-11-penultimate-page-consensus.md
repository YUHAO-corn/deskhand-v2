# 倒数第二页：三个维度的判断

## 定位

夹在 Clipboard 架构页和 Closing Thought（产品哲学）之间。
前面三个功能模块已经证明了"能做"，这一页要证明"能想"——做完之后能提炼出可迁移的产品思考框架。

面试官看完这一页的感受应该是：他不只是能做功能，还能从实践中抽象出可复用的产品判断，而且这个判断可以直接带到下一个桌面智能体或 AI 产品项目里用。

## 文案

### 标题

做完 Deskhand，我认为桌面智能体的能力取决于三件事

## 概念表述（完整文字，用于理解语境，不直接上页面）

做 Deskhand 的过程中，我在三个方向上分别做了探索——Gen UI、Skill System、Clipboard。

Gen UI 解决的是意图理解的问题。用户很多时候不是不知道自己要什么，而是说不清楚。长 prompt 写不好，写好了模型也不一定理解对。我做的事情是给出自然语言之外的交互方式，让用户可以通过选择、比较、探索来表达意图，而不是只能靠打字。

Clipboard 解决的是资源触达的问题。AI 现在能调用的资源越来越多——MCP 工具、本地文件、沙盒环境——但用户日常工作中最密集的上下文其实在剪贴板里，而系统默认根本不会收集剪贴板历史。我先给 Deskhand 搭建了一套剪贴板记录的能力，然后把这层历史数据开放给 Agent，让它能拿到用户刚刚处理过的东西，而不是每次都要用户重新解释一遍。

Skill System 解决的是能力扩展的问题。Skill 生态已经存在了，但对非技术用户来说，发现、搜索、安装、创建这条链路全要自己走，门槛太高。我把这件事交给了 Agent——让它自己去观察用户的工作模式，自己去找合适的 skill，自己去安装；如果找不到合适的，它也能自己创建。用户不需要知道 skill 是什么，只需要发现 Agent 越来越好用了。

这三个方向加在一起，其实是我对"桌面智能体现阶段该往哪走"的一个判断：意图理解、资源打通、编排调度，这三件事决定了一个桌面智能体能做到什么程度。意图理解决定方向对不对，资源打通决定能不能走到终点，编排调度决定路上的质量和速度。Deskhand 目前在每个维度上都只是一个切片，但每个切片都验证了这个方向是有价值的。

而且每个维度都还有很大的空间可以继续展开。比如意图理解上，除了 Playground 和 This or That，还可以做 brainstorm 画布、问答漏斗这些新的交互模式；资源打通上，剪贴板只是第一步，日历、提醒事项、备忘录这些 OS 级资源都可以接入；编排调度上，我最想做的是长期记忆专家——当你和一个 Agent 协作完成了一个任务，而且合作得很好，你应该能把它保存下来，让它记住你的偏好和风格，下次类似的任务直接交给它。

### 设计说明

1. **递进箭头链**：三个维度之间有依赖关系（意图理解是前提，资源打通在方向对的基础上才有意义，编排调度是前两者都成立后的上限），所以用递进箭头而不是并列。每个节点下方附一句"决定了什么"。

2. **三列卡片**：与箭头链上的三个节点一一对齐。每张卡片内部分两层——"已验证"是 Deskhand 已经做了的，"可展开"是基于这个框架可以继续推导出的下一步。两层之间用视觉分隔区分。

3. **箭头链和卡片的关系**：箭头链是抽象框架，卡片是具体实践。上面是判断，下面是证据。
